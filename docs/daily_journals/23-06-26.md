Section 2 – Failed Attempt

Initially I wrote most of the constructors separately and every constructor repeated the same initialization code. Then I thought I could simply use clear() for that, but that turned out to be a bad idea because clear() is supposed to free existing memory, not initialize a fresh object. It even introduced memory leak issues.

Another mistake I made was resetting size_ before destroying the elements inside clear(). Since the size became zero first, my destructor loop never actually executed.

For resize, I was multiplying the capacity by two without thinking about the case when capacity itself was zero. That kept the capacity at zero forever and eventually resulted in the illegal instruction during testing.

I also originally made functions like append() and set() accept pointers or non-const references where they really shouldn't. That created unnecessary compilation errors while testing with strings and literals.

Section 5 – Learning Reflection

Today was probably the first time manual memory management actually started making so much sense to me.

Earlier I was thinking that free() or delete would automatically clean everything, but while using placement new I understood that object destruction and memory deallocation are actually two different steps. First every object's destructor has to run, then the raw memory can be released.

I also realised that helper functions are not just for making the code shorter. Functions like allocate(), construct(), destroyobj() and init() made the implementation much easier to maintain because I only had to fix logic in one place instead of updating the same code everywhere.

Valgrind was also more useful than I expected. Some bugs didn't crash the program at all, but Valgrind immediately pointed out invalid writes and memory issues that I would've probably missed otherwise.

Another thing I learnt was not to assume my resize logic works just because normal test cases pass. Edge cases like zero capacity or shrinking the array exposed bugs that weren't obvious at first.

By the end of the day the Dynamic Array implementation was mostly complete and tested, and I had also finished the basic implementation of the Singly Linked List with clean memory management and no leaks reported by Valgrind.
