At first I thought something was wrong in my linked list implementation because the exception was coming from there, but the actual bug ended up being inside the HashMap itself.

Section 2 – Failed Attempt

I spent quite a while debugging the linked list because the exception clearly pointed towards it. I checked remove(), operator[], pointer traversal, and even tested the linked list separately. Everything looked fine.

After going through the HashMap line by line again I finally noticed I had accidentally used buckets instead of bucket in one place. Because of that I was accessing the wrong linked list and it eventually threw the invalid index exception. That bug took way longer to find than it should have.

While designing the HashMap I also first planned to use prime bucket sizes for resizing. I even started thinking about maintaining a prime array, finding the next prime and writing helper functions for it. After some time I realised it was making the implementation unnecessarily complicated, so I scrapped that idea and switched to bucket sizes that are powers of two. It simplified both resizing and rehashing.

Section 5 – Learning Reflection

Today was less about writing a lot of code and more about figuring out how different data structures fit together.

While making the HashMap I realised that building reusable data structures actually saves a lot of work later. Instead of writing bucket logic from scratch, I could directly use my Dynamic Array to store buckets and each bucket itself was just another Singly Linked List. That was kind of the point of making these libraries separately.

I also learnt that compiler errors are usually easier than logic bugs. The INVALID INDEX exception wasted a lot of time because I kept debugging the linked list even though the real mistake was in the HashMap. It reminded me that the place where an error appears is not always where it starts.

Another thing I changed today was the overall design. Prime-sized buckets are theoretically nice, but for this project they would've needed extra helper functions and more bookkeeping than I wanted. Using powers of two made the implementation much cleaner and easier to maintain.

By the end of the day my linked list testing was mostly complete, the basic HashMap structure was ready, constructors and helper methods were tested with Valgrind, and the main methods like set(), get(), remove(), clear() and exists() were implemented. Rehashing and hash functions were the main things left to finish next.
